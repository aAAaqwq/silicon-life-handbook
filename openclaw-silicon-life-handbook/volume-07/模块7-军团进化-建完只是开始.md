# 卷五｜军团协同：从单体到组织的跃迁
## 模块7｜军团进化
### ——为什么军团不是“建完就完”，而是必须持续进化

---

## 一句话结论

> **军团进化解决的不是“建不建”，而是“建完后如何避免腐化、如何持续升级、如何防止组织熵增”。**

换句话说：

> **没有进化机制，军团一定会腐化；有了进化机制，军团才能持续升级。**

---

## 这段在训练什么

本模块训练的是：

1. **如何理解军团进化的系统价值**
2. **如何建立进化机制**
3. **如何防止军团腐化**

如果一个人以为军团建完就完事了，他不懂军团；
真正懂的人会知道：

> **军团建完只是开始，真正的挑战是持续进化。**

---

## 为什么军团必须进化

很多人建完军团后就以为大功告成。

这会带来一个致命问题：

> **组织一定会腐化，除非有进化机制。**

### 没有进化机制会发生什么

#### 问题1：角色固化
表现：
- agent 长期不升级
- 能力停滞

后果：
- 跟不上业务发展
- 逐渐被淘汰

#### 问题2：协议过时
表现：
- 老协议不适应新场景
- 没人更新

后果：
- 协议失效
- 协同混乱

#### 问题3：治理失灵
表现：
- 账本长期不审计
- 奖惩失效

后果：
- 干好干坏一个样
- 优秀 agent 流失

#### 问题4：信任衰减
表现：
- 信任不更新
- 过度信任或信任不足

后果：
- 授权不合理
- 出问题

#### 问题5：协同失效
表现：
- handoff 不顺
- 让位规则过时

后果：
- 掉棒增加
- 效率下降

所以进化机制的本质不是“锦上添花”，而是：

> **防止组织腐化的基础设施。**

---

## 军团进化的四大机制

### 机制1：定期审计

#### 审计内容
1. 角色是否仍然合理
2. 边界是否仍然清晰
3. 协议是否仍然有效
4. 治理是否仍然有效
5. 信任是否仍然合理

#### 审计周期
- 日常审计：每天
- 周度审计：每周
- 月度审计：每月
- 重大变更后审计：立即

#### 审计产物
- 审计报告
- 问题清单
- 整改建议

---

### 机制2：协议升级

#### 升级触发条件
1. 审计发现问题
2. 业务场景变化
3. agent 能力提升
4. 外部环境变化

#### 升级流程
1. 识别需要升级的协议
2. 设计新版本
3. 在测试环境验证
4. 灰度发布
5. 全量发布
6. 废弃旧版本

#### 升级记录
```md
# 协议升级记录

## 升级 #1
- 协议：让位协议 v1.0 → v1.1
- 原因：发现抢答增加
- 变更：新增“让位必须在 10 秒内表达”
- 验证：测试环境跑 50 轮，无抢答
- 发布时间：2026-03-29
```

---

### 机制3：角色升级

#### 升级条件
1. 信誉分达到阈值
2. 累计军功达到阈值
3. 连续 N 轮无失误
4. 通过升级测试

#### 升级流程
1. agent 达到升级条件
2. 提交升级申请
3. 审核账本
4. 设计升级测试
5. 执行测试
6. 审批通过
7. 授权升级
8. 记录升级

#### 角色升级记录
```md
# 角色升级记录

## xuanyuan 技术agent 升级
- 原等级：锐士
- 新等级：精锐
- 升级原因：信誉分 92，连续 20 轮无失误
- 升级测试：通过
- 新增权限：
  - 可独立决策技术方案
  - 可直接 handoff 给其他 agent
- 升级时间：2026-03-29
```

---

### 机制4：组织扩编

#### 扩编触发条件
1. 业务规模扩大
2. 专业域增加
3. 现有 agent 负载过高
4. 新业务场景出现

#### 扩编流程
1. 识别扩编需求
2. 设计新角色
3. 定义边界
4. 设计协同协议
5. 训练新 agent
6. 灰度接入
7. 全量接入
8. 更新军团原型

#### 扩编记录
```md
# 组织扩编记录

## 扩编 #1
- 时间：2026-03-29
- 原因：新增电商业务线
- 新增角色：qilin（电商 agent）
- 边界：电商相关问题
- 协同协议：handoff 给 tiangong（产品设计）
- 训练：通过 L1→L2 训练
- 接入：灰度 3 天，全量接入
```

## 军团进化的常见问题

### 问题1：只建不审计
表现：
- 建完就不管
- 定期审计缺失

后果：
- 问题积累
- 最后大崩盘

### 问题2：协议不升级
表现：
- 协议过时
- 没人更新

后果：
- 协同失效
- 到处掉棒

### 问题3：角色不升级
表现：
- agent 能力停滞
- 没有升级机制

后果：
- 跟不上业务
- 优秀 agent 流失

### 问题4：扩编随意
表现：
- 想加就加
- 没按流程

后果：
- 组织混乱
- 边界模糊

---

## 军团进化最佳实践

### 实践1：建立定期审计机制
- 每天/每周/每月审计
- 重大变更后立即审计

### 实践2：协议版本化管理
- 每个协议都有版本号
- 升级有记录
- 可回滚

### 实践3：角色升级制度化
- 明确升级条件
- 有升级测试
- 有升级记录

### 实践4：扩编按流程
- 不随意扩编
- 按流程扩编
- 有扩编记录

### 实践5：进化记录公开
- 进化记录对军团内公开
- 可追溯
- 可学习

---

## 训练题

### 训练题1
为什么军团必须进化？

**标准答案：**
因为组织一定会腐化，除非有进化机制。

### 训练题2
进化四大机制是什么？

**标准答案：**
定期审计 + 协议升级 + 角色升级 + 组织扩编。
