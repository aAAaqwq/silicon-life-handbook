# 卷五｜军团协同：从单体到组织的跃迁
## 模块1｜路由系统
### ——为什么“谁该答”不是礼貌问题，而是军团秩序的第一道闸门

---

## 一句话结论

> **路由系统解决的不是“能不能答”，而是“谁该先答、谁不该答、谁该让位、谁该兜底”。**

换句话说：

> **没有路由系统，军团就会退化成抢答团；有了路由系统，军团才能变成有秩序的组织。**

---

## 这段在训练什么

本模块训练的是：

1. **如何理解路由在军团中的系统价值**
2. **如何设计确定性路由规则**
3. **如何避免“看起来有路由，实际靠抢答”的混乱**

如果一个人以为路由只是“消息分发”，他不懂军团；
真正懂的人会知道：

> **路由是组织秩序的第一道闸门，决定了军团会不会变成菜市场。**

---

## 为什么路由系统是军团第一优先级

很多人建军团时会先想：
- 每个 agent 要有什么能力
- 每个 agent 的 prompt 怎么写
- 每个 agent 的名字叫什么

但这些都比不上一个更底层的问题：

> **一条消息进来了，谁该先接？**

### 没有路由系统会发生什么

#### 混乱1：抢答
表现：
- 一条消息进群
- 三个 agent 同时回答
- 用户不知道听谁的

#### 混乱2：失答
表现：
- 一条消息本该某个专家答
- 结果被默认 agent 吞了
- 专家根本没看到

#### 混乱3：串角色
表现：
- 技术问题被战略 agent 抢答
- 内容问题被技术 agent 乱接
- 边界模糊

#### 混乱4：责任不清
表现：
- 出了问题不知道该找谁
- 没有明确责任人
- 无法追责

所以路由系统的本质不是“分发技术”，而是：

> **把组织秩序写进系统规则里。**

---

## 路由系统的四层结构

### 第一层：渠道级路由
规则：
- 哪个渠道归哪个 agent
- 哪个群归哪个 agent
- 哪个私聊归哪个 agent

例子：
- 产品群 → 产品 agent
- 技术群 → 技术 agent
- 老板私聊 → 战略 agent

### 第二层：账号级路由
规则：
- 哪个 accountId 归哪个 agent
- 同一渠道不同账号如何区分

例子：
- 账号A → 客服 agent A
- 账号B → 客服 agent B

### 第三层：对等方级路由
规则：
- 哪个用户归哪个 agent
- 同一群里不同人如何区分

例子：
- VIP 用户 → 贴身 agent
- 普通用户 → 通用 agent

### 第四层：提及/关键词路由
规则：
- @某个 agent 时归谁
- 提到某个关键词时归谁

例子：
- @技术 → 技术 agent
- 提到“合同” → 法务 agent

---

## OpenClaw 的路由优先级（官方机制）

根据 `concepts/multi-agent.md`，OpenClaw 的绑定路由遵循：

> **最具体优先**

优先级大致是：
1. **peer 匹配**（最具体）
2. **guildId / teamId**
3. **accountId**
4. **channel 级匹配**
5. **默认 agent**（兜底）

这意味着：
- 对等方绑定 > 渠道绑定
- 渠道绑定 > 默认绑定

## 路由系统常见设计错误

### 错误1：只有默认路由，没有细分
表现：
- 所有消息都进同一个 agent

后果：
- 不是军团，只是单 agent 多入口

### 错误2：路由规则冲突
表现：
- 多条规则同时匹配
- 不知道谁该答

后果：
- 实际退化成抢答

### 错误3：路由规则过细
表现：
- 每个关键词都配一条规则
- 规则数量爆炸

后果：
- 难以维护
- 实际靠抢答

### 错误4：没有兜底机制
表现：
- 消息没有被任何规则匹配
- 无人响应

后果：
- 消息丢失
- 用户体验差

---

## 路由系统最佳实践

### 实践1：先粗后细
- 先建立渠道级路由
- 再逐步细化到关键词

### 实践2：明确优先级
- 写清楚规则的优先级顺序
- 避免冲突时不知道听谁的

### 实践3：必须有兜底
- 每个渠道必须有默认 agent
- 未被匹配的消息不能丢失

### 实践4：定期审计
- 定期检查路由规则是否仍然合理
- 清理过期规则

### 实践5：记录路由日志
- 记录每条消息被路由到哪个 agent
- 出问题时可以追溯

---

## 路由系统的验收标准

### 验收1：无抢答
- 一条消息只会被一个 agent 先接
- 除非明确配置了并行路由

### 验收2：无失答
- 每条消息都有明确的接收 agent
- 没有消息被丢失

### 验收3：可追溯
- 可以查到每条消息被路由到谁
- 可以判断路由是否正确

### 验收4：可维护
- 路由规则清晰、不过度复杂
- 新增 agent 时规则易扩展

## 训练题

### 训练题1
为什么路由系统是军团第一优先级？

**标准答案：**
因为没有路由，军团就会退化成抢答团，无法形成组织秩序。

路由系统四层结构是什么？

**标准答案：**
渠道级 → 账号级 → 对等方级 → 提及/关键词级。

**标准答案：**

## 验收口径

你判断这个模块有没有学会，只看三件事：

### 1）能否设计路由规则
不是“大概知道”，而是能写出四层路由规则表。

### 2）能否避免路由冲突
能说清楚规则优先级，不会出现抢答。

### 3）能否保证无失答
有兜底机制，每条消息都有明确的接收者。
